## My First Week Learning GitHub (And Why git pull origin Was So Annoying

For this week, setting up a staging area was actually pretty simple. I ended up using Google Docs for it. I know it mentioned that I didn’t really need to use a Google document, but honestly, I still went with it because it’s just easier to organize my thoughts there.

What I really wanted to focus on this week was getting more comfortable with GitHub and VS Code. I ran into some issues, especially with one command: `git pull origin`. That one gave me more trouble than I expected.

I’ll be honest—it was pretty annoying.

The main problem is that if you don’t check things properly (like using `git commit -v`), you might not realize which repository you’re actually working with in VS Code. That means you could end up pulling from the wrong place without noticing.

So yeah, lesson learned: always double-check your fetch/repo before running `git pull origin`. It saves you from a lot of confusion later.
